# i32-Red-Black-Tree

用纯 C 语言从零实现的红黑树，包含插入、删除、旋转和修复的全部代码。

# 前言

实现时，理解每一步的用意和解决思路，各种情况的判断和旋转特性。

## 插入修复的核心

```c
RBNode *grand = parent->parent;  // 锚点，旋转前后不变
...
parent = grand->left;  // 直接从grand赋值，更稳
```

## 核心理解：删除修复到底在修什么？

### 概念 1：红色是“潜在”的黑色

> 就是看兄弟那边有没有红色，然后把红色弄到删除的那一边，然后把红色变成黑色。因为红色少一个多一个无所谓，只要没有两个红色链接就行了，这样就平衡了。就相当于：红色是潜在的黑色，红色转换成黑色 = 加一个黑色。删除一个黑色 + 加一个黑色（红变黑）= 没变。

**整个删除修复，就是在找一个能染黑的红色。**

### 概念 2：旋转的本质是“节点归属权”的转移

> 就是让 parent 变成子。parent 原本是公共祖先，不算某个分支独有的节点，但变成子后，它本身就属于这条分支的一个节点（不管 parent 原来是黑还是红，变成子后都被强制染黑，相当于加一个黑色）。然后远端少一个黑，把红色变成黑就抵消了。左边原本少了一个黑，由 parent 变成子后补上了；右边红色变成黑色也抵消了。最后两个分支黑色数量对等！

**旋转不是魔法，是改变节点的归属关系。**

### 概念 3：循环的本质是“层层上报”的亏空接力

> 假如 parent->right 那分支原本有 8 个黑，左边也有 8 个黑。然后我删除了右边分支下面的某个黑色节点（变成 7 个黑了）。兄弟全黑，处理不了，兄弟染红一起亏，问题上移给 parent。此时 parent 是红色，while 循环退出，parent 染黑。这个时候右边又是 8 个黑了……好像平衡了！！

**while 循环就是在往上找红色。找到就染黑收工，到根还没找到，整棵树降一级。**
